# /galchan-script — 台本生成（2026-03-26 v2）

ネタ候補から、**競合分析の結果を直接反映**した台本を生成する。
リサーチ → トーン吸収 → 構造分析 → 構成決定 → 台本生成 → バリデーション → 保存。

> **ターゲット:** 40代中年女性。お金を無駄にしたくない、損したくない、後悔したくない層。

## 引数
- 引数あり: `$ARGUMENTS` をテーマとして使用
- 引数なし: ネタ候補リストの★★★1位を自動選択
- パイプライン経由: `/galchan-ideas` の出力からテーマ+スコアリング根拠が渡される場合あり

## Step 4: 台本生成（ディレクターが直接実行）

> **重要**: サブエージェントではなくメインが直接生成する。
> **Step 1のリサーチ結果（内容）+ Step 1.5のトーンメモ（書き方）の両方を参照しながら書く。**

### フォーマット（台本生成ルール.md準拠）
- ヘッダー行なし。1行目「ナレーション」、2行目「タイトル（10文字程度）」
- タブ区切り3列。コロン不要。かぎ括弧不要
- 1行30字以上。基本35字、たまに70字

### イントロ（ガルこめパターン＋属人性）
```
ナレーション	皆さんこんにちは！今回は、
タイトル	（10〜15文字のテーマ）
ナレーション	についてまとめました。（私も〜の体験一言）。それではいってみよう！！
```
- **合計3行。10秒以内で本題に入る**
- 「私も〜」は**消費者・視聴者と同じ目線**（✅「私も知らずに買ってた」 ❌「私も調べてみて」）
- コメント誘導はエンディングに移動

### トーンの基準（競合分析から。自チャンネル台本ではない）
- **体験談が主役、数値は補足**（「教科書を読み上げる先生」ではなく「ママ友の井戸端会議」）
- **「なんで買ったんだろう」「高かったから捨てる勇気もない」**のような自問・葛藤型の発言
- **「うちも〜」で始まる共感の連鎖**。共感→別の体験→さらに共感のループ
- **短い感情表現**（「無駄にでかくて邪魔。」「本当に無駄だった。」）
- **説明しない**。「〇〇には△△という成分が含まれており…」ではなく「〇〇食べてたけど成分表見てビビった」
- **権威機関名は台本全体で2-3回まで**。最初に1回言えば十分。**公的データがないテーマでは無理に入れない**
- **専門用語は使わない**。使う必要がある場合は日常の言葉で言い換える（例：「トランス脂肪酸」→「マーガリンに入ってる体に悪い油」、「アクリルアミド」→「焦げに含まれる発がん性物質」のように補足する）

### 数値目標
- 本文文字数 9,000〜10,000字
- 行数 285〜350行
- イッチ比率 **冒頭1回のみ（5%以下）**

### 構成
- **トピック数はStep 2で決定した数**
- **トピック間はぶつ切り**（接続語なし。「あと」「そういえば」禁止）
- **後半ほどテンポを上げる**
- **感情ピークは前半に配置**

### エンディング（自チャンネル固定文を使用）
- 前向きなまとめ（2-3行。具体的行動を含む）
- コメント誘導（1行。具体的な問いかけ）
- 固定文（一字一句変更禁止）:
```
ナレーション	少しでも参考になったらチャンネル登録、高評価をお願いします。
ナレーション	最後までご視聴ありがとうございました。
```

### 商品リスト
- 商品名が登場する台本 → スプシの隣シートに商品一覧を別途作成

### 生成手順
1. **イントロを書く**（3行）
2. **前半トピック（全体の1/3）を1トピックずつ書く** → 引きの強いものから
3. **中盤トピック（全体の1/3）を書く** → テンポアップ。軽い話題を1つ挟む
4. **後半トピック（全体の1/3）を書く** → 圧縮
5. **エンディングを書く**（8-10行、コメント誘導含む）
6. 全体を結合してTSV完成

> 各トピック生成後に行数・文字数をカウントして進捗を報告する

### 台本生成品質ゲート（Step 5に進む前の確認）
- [ ] 本文文字数が9,000〜10,000字の範囲内
- [ ] 行数が285〜350行の範囲内
- [ ] イッチが冒頭1回のみ
- [ ] トピック数がStep 2で決定した数と一致
- [ ] エンディング固定文が一字一句正確

### 7-1. タイトル案（3つ）

- 「40代」キーワード含む
