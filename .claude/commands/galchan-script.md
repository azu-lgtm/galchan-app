ネタ候補から高品質な台本を生成する。リサーチ3エージェント並列 → メインが直接台本生成 → 自動バリデーション → 保存。

## 引数
- 引数あり: `$ARGUMENTS` をテーマとして使用
- 引数なし: ネタ候補リストの★★★1位を自動選択

## Step 1: リサーチフェーズ（3エージェント並列）

**Agent tool で以下の3エージェントを同時に起動する（必ず並列）：**

### Agent A: ガルちゃんリサーチャー
テーマに関連するガルちゃんトピックを調査。WebSearchで「ガルちゃん {キーワード}」検索 → 上位3トピックをWebFetchで取得 → プラス票TOP20・よく使われるフレーズ・リアルな体験談・40代主婦の共感コメントを抽出 → JSON形式で返す。

### Agent B: 公的機関リサーチャー
テーマに関連する公的機関データを収集。農林水産省・国民生活センター・消費者庁等をWebFetch（403ならWebSearchで代替）→ 事故件数・具体的商品名・注意喚起内容・推奨代替策を抽出 → JSON形式で返す。

### Agent C: 競合差別化アナリスト
競合チャンネルリスト.md + 勝ちパターン.md + アナリティクス.md + 過去台本一覧を読み込み、WebSearchで「{テーマ} ガルちゃん まとめ YouTube」を検索 → 競合アプローチ・差別化ポイント・避けるべき切り口・タイトル/サムネ案を分析 → JSON形式で返す。

## Step 2: 構成決定（ディレクター）

3エージェントの結果を統合して以下を決定：

1. **6トピックのテーマ**（ガルちゃん票数順に優先）

5. **サムネ案3つ**（上段6文字以内/下段15文字以内）

## Step 3: 台本生成（ディレクターが直接実行）

> **重要**: サブエージェントではなくメインが直接生成する。
> サブエージェントに350行のTSV生成を任せると行数不足・フォーマット崩れが頻発するため。

### 台本生成ルール（台本生成ルール.mdから抜粋・厳守）

**台本生成ルール.md（唯一の正）を全て読み込んでから生成すること。**
以下は要点のみ。詳細は台本生成ルール.mdを参照。

■ フォーマット:
- ヘッダー行なし。1行目「ナレーション」、2行目「タイトル（10文字程度）」
- タブ区切り3列。コロン不要。かぎ括弧不要
- 1行40〜70字。句読点を適切に入れる

■ イントロ:
- 挨拶→タイトル→体験談3行→コメント誘導→「それではいってみよう！！」
- 体験談で具体的な年齢は入れない（設定ブレ防止）

■ 数値目標:
- 本文文字数 8,600〜9,000字
- 行数 285〜350行
- イッチ比率 10%以下

■ 会話:
- 同じ話者の連続発言禁止
- 1発言に「状況/行動/結果」のうち2つ以上
- 3〜5発言ごとに空気を変える
- 権威機関名は台本全体で2〜3回まで
- 専門用語は最小限

■ 商品リスト:
- 商品名が登場する台本 → スプシの隣シートに商品一覧を別途作成

### 生成手順（必ず1トピックずつ）

1. **イントロを書く**（5〜7行）
2. **トピック1を書く**（45〜55行） → 使った表現をメモ
3. **トピック2を書く** → 前トピックの表現リストを確認してから書く
4. **トピック3〜6を同様に繰り返す**
5. **エンディングを書く**（10〜12行）
6. 全体を結合してTSV完成

> 各トピック生成後に行数をカウントして進捗を報告する

## Step 4: 自動バリデーション

生成したTSVをバリデーションスクリプトでチェック：
```bash
python C:\Users\meiek\Desktop\ClaudeCode-projects\galchan-app\local-tool\validate_script.py "TSVファイルパス"
```

- **passed=true** → Step 5へ
- **has_critical=true** → 該当箇所を修正して再検証（最大3回ループ）
- **軽微なissueのみ** → 修正してStep 5へ

## Step 5: ファイナルレビュー（レビュアーエージェント）

**Agent tool でレビュアーを起動：**

done台本（先頭50行）と生成台本を比較し、以下をチェック：
- 会話の自然さ（done台本と同レベルか）
- トピック切り替えの自然さ
- スレ民6人の発言分布
- 専門用語の使用頻度
- 1発言あたりの長さ
- 100点満点でスコアリング（90+: そのまま保存、70-89: 軽微修正、70未満: 書き直し）
