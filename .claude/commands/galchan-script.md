## Step 1: リサーチフェーズ（3エージェント並列）— 「何を書くか」を集める

> **台本の中身（ネタ・事実・体験談・差別化）を集めるフェーズ。**
> ネタ出し（/galchan-ideas）は別スキル。ここではテーマが決まった後の深掘りリサーチ。

**Agent tool で以下の3エージェントを同時に起動する（必ず並列）：**

### Agent A: ガルちゃんリサーチャー — 体験談・リアルな声を集める
WebSearchで「ガルちゃん {キーワード}」検索 → 上位3トピックをWebFetchで取得。
**拾うもの:**
- プラス票TOP20の投稿（＝視聴者が最も共感する内容）
- リアルな体験談（具体的なエピソード。商品名・金額・期間が入っているもの優先）
- よく使われるフレーズ（台本のセリフに直接使える表現）
- 40代主婦の共感コメント（ターゲットの本音）
- **意見が割れているポイント**（台本内の掛け合いに使える）
→ JSON形式で返す

### Agent B: 事実・データリサーチャー — 台本の根拠を集める
テーマに関連する事実データをWebSearch/WebFetchで収集。
**拾うもの:**
- 具体的な数値（成分量・事故件数・価格・期間等）
- 商品名・ブランド名のリスト
- 代替策・正しい使い方
- 公的機関のデータ（**ある場合のみ。なくてもOK**）

**公的機関の引用ルール:**
- テーマに関連する公的機関データが**ある場合のみ**収集する。ないテーマも多い
- 「〇〇が警告している」→ 実際の警告文書・ガイドラインが確認できた場合のみ
- 確認できなかった情報は「未確認」とマークし、台本に入れない
- 数値は公式発表をそのまま使用。丸め・誇張・推測禁止
- **台本全体で公的機関名は2-3回まで**。多いと解説番組になる
- **捏造・推測での機関名引用は絶対禁止**
- **公的データがないテーマでも問題ない** — 体験談ベースで十分成立する
→ JSON形式で返す

## Step 2: 構造分析（競合ベースのトピック数・テンポ決定）

勝ちパターン統合版 + 競合台本構造分析を参照し、以下を決定する：

1. **テーマタイプの判定**（注意喚起系 / 商品紹介系 / マナー系 / 健康美容系）
2. **最適トピック数の決定**（テーマタイプ別の推奨値）
   - 注意喚起・逆説系: 12-17トピック（1トピック500-800字）
   - 商品紹介・比較系: 25-35トピック（1トピック200-350字）
   - マナー・常識系: 15-20トピック（1トピック400-700字）
   - 健康・美容系: 10-15トピック（1トピック700-1,000字）
3. **テンポ設計**
   - 前半（1/3）: 1トピック1分〜1分半（丁寧に導入）
   - 中盤（1/3）: 1トピック45秒〜1分（テンポアップ）
   - 後半（1/3）: 1トピック30秒〜45秒（圧縮）
4. **感情ピークの配置**（前半に引きの強いトピック）
5. **感情動線パターンの選択**
   - 注意喚起系: 驚き→後悔→あるある→怖い→得した感 ループ
   - 商品系: 後悔→比較→納得→驚き→得した感 ループ
   - マナー系: 恥→共感→恐怖→安堵→恥 ループ

## Step 3: 構成決定（ディレクター）

3エージェントの結果 + 構造分析を統合して以下を決定：

1. **トピック一覧**（Step 2で決めた数。ガルちゃん票数順に優先）
2. **各トピックで使う具体データ**（事実データ紐づけ。公的機関データは**あれば**検証済みのみ）
3. **各トピックの配置**（引きの強いものを前半に）
4. **感情動線**（テーマタイプに合った感情パターン）
5. **タイトル案3つ**
   - テンプレ: 【感情ワード】＋行動描写＋…（余韻）＋衝撃の結果＋【40代/権威機関】
   - サムネとタイトルの情報重複ゼロ（自チャンネルCTR 1.9%の失敗を繰り返さない）
   - メインキーワードは広いカテゴリ（「食品」「美容」等）にする
6. **サムネ案3つ**（競合網羅分析のパターンを適用）
   - サムネ=感情、タイトル=内容
   - 括弧内に3-4つのサブポイント
   - 命令形 or 直接攻撃

→ **構成案を提示して承認を得る**

## Step 4: 台本生成（ディレクターが直接実行）

> **重要**: サブエージェントではなくメインが直接生成する。
> **Step 1のリサーチ結果（内容）+ Step 1.5のトーンメモ（書き方）の両方を参照しながら書く。**

### フォーマット（台本生成ルール.md準拠）
- ヘッダー行なし。1行目「ナレーション」、2行目「タイトル（10文字程度）」
- タブ区切り3列。コロン不要。かぎ括弧不要
- 1行30字以上。基本35字、たまに70字

### イントロ（自チャンネル固定文を使用）
```
ナレーション	皆さんこんにちは！今回は、
タイトル	（10文字程度の短いテーマ）
ナレーション	（逆説の1文：常識を覆す具体的な事実。15-25字）
ナレーション	それではいってみよう！！
```
- **10秒以内で本題に入る（3行以内）**
- コメント誘導はエンディングに移動

### トーンの基準（競合分析から。自チャンネル台本ではない）
- **体験談が主役、数値は補足**（「教科書を読み上げる先生」ではなく「ママ友の井戸端会議」）
- **「なんで買ったんだろう」「高かったから捨てる勇気もない」**のような自問・葛藤型の発言
- **「うちも〜」で始まる共感の連鎖**。共感→別の体験→さらに共感のループ
- **短い感情表現**（「無駄にでかくて邪魔。」「本当に無駄だった。」）
- **説明しない**。「〇〇には△△という成分が含まれており…」ではなく「〇〇食べてたけど成分表見てビビった」
- **権威機関名は台本全体で2-3回まで**。最初に1回言えば十分。**公的データがないテーマでは無理に入れない**
- **専門用語は使わない**。使う必要がある場合は日常の言葉で言い換える（例：「トランス脂肪酸」→「マーガリンに入ってる体に悪い油」、「アクリルアミド」→「焦げに含まれる発がん性物質」のように補足する）

### 数値目標
- 本文文字数 9,000〜10,000字
- 行数 285〜350行
- イッチ比率 **冒頭1回のみ（5%以下）**

### 構成
- **トピック数はStep 2で決定した数**
- **トピック間はぶつ切り**（接続語なし。「あと」「そういえば」禁止）
- **後半ほどテンポを上げる**
- **感情ピークは前半に配置**

### エンディング（自チャンネル固定文を使用）
- 前向きなまとめ（2-3行。具体的行動を含む）
- コメント誘導（1行。具体的な問いかけ）
- 固定文（一字一句変更禁止）:
```
ナレーション	このチャンネルでは40代からの失敗回避をテーマに
ナレーション	ガールズチャンネルや私の体験談を交えてお話ししています。
ナレーション	少しでも参考になったらこれからも失敗回避の話を続けていくので
ナレーション	そして動画下のサンクスでの応援もいただけると嬉しいです!!
ナレーション	最後までご視聴ありがとうございました。
```

### 商品リスト
- 商品名が登場する台本 → スプシの隣シートに商品一覧を別途作成

### 生成手順
1. **イントロを書く**（4行）
2. **前半トピック（全体の1/3）を1トピックずつ書く** → 引きの強いものから
3. **中盤トピック（全体の1/3）を書く** → テンポアップ。軽い話題を1つ挟む
4. **後半トピック（全体の1/3）を書く** → 圧縮
5. **エンディングを書く**（8-10行、コメント誘導含む）
6. 全体を結合してTSV完成

> 各トピック生成後に行数・文字数をカウントして進捗を報告する
