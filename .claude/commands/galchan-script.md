# /galchan-script — 台本生成→TSV出力

ネタ候補から台本を生成し、YMM4用TSVファイルとして出力します。

## 引数
- 引数あり: `$ARGUMENTS` をテーマとして使用（例: `/galchan-script 国民生活センター警告×日用品`）
- 引数なし: ネタ候補リストの★★★1位を自動選択

### 3. 台本生成ルール

**チャンネルコンセプト**: 失敗回避専門（買って後悔・やって後悔・知らなくて後悔）
**ターゲット**: 40代女性（主婦・パート）
**動画尺**: 20〜25分想定
**総行数**: 必ず280〜320行になるように生成すること（少なすぎは絶対NG）
- イントロ: 10〜15行
- 本編: 250〜270行（エピソード6本 × 各40〜45行）
- エンディング: 10〜15行

**キャラクター（話者）と役割**:
- `ナレーター` — イントロ・エンディング・シーン説明・エピソード導入
- `イチコ` — スレ主。体験談・問題提起・追加情報（各エピソード5〜8行）
- `スレ民1` — 驚き・共感型（「えっ、知らなかった！」「ウソでしょ！？」）
- `スレ民2` — 体験談型（「うちも同じことあった」「実際にやってしまった」）
- `スレ民3` — 追加情報・知識型（「それって〜だから危ないんだよね」）
- `スレ民4` — 疑問・心配型（「じゃあどうすればいいの？」「代わりに何を使えば？」）
- `スレ民5` — まとめ・納得型（「勉強になった」「これは広めなきゃ」）

**重複・単調防止ルール（必ず守ること）**:
- 同じスレ民が連続して2回以上発言しない
- 「怖い」「知らなかった」「ありがとう」などの単語を同一エピソード内で3回以上繰り返さない
- 各スレ民のセリフは毎回異なる切り口・感情で書く
- エピソードをまたいで同じようなやりとりのパターンを繰り返さない
- イチコの発言は具体的な数字・事例・農林水産省の実際の警告内容を含める

**構成**:
```
[イントロ: 10〜15行]
ナレーター: 「皆さんこんにちは！今回は〜についてご紹介します」
ナレーター: タイトルコール（タイトルをそのまま読む）
ナレーター: テーマの概要・なぜ重要かの説明（2〜3行）
ナレーター: 「それでは見ていきましょう！」

[シーン転換]（話者=空欄、本文=「（シーン転換）」、SE=「シーン転換」）

[本編エピソード × 6本: 各40〜45行]
各エピソードの構成：
  ナレーター: 「▼ 第N話：〜」エピソード導入（1〜2行）
  イチコ: 問題提起・体験談（5〜8行）
  スレ民1: 驚き・共感（2〜3行）
  スレ民2: 自分の体験談（2〜3行）
  イチコ: 詳細・農林水産省の具体的な警告内容（3〜5行）
  スレ民3: 追加情報・理由の説明（2〜3行）
  スレ民4: 疑問・対策を聞く（1〜2行）
  イチコ: 対策・正しい方法を説明（3〜4行）
  スレ民5: まとめ・納得（1〜2行）
  スレ民1〜4: 追加コメント（各1〜2行、計4〜6行）
  [シーン転換]

[エンディング: 10〜15行]
ナレーター: 各エピソードのまとめ（エピソードごとに1行ずつ）
ナレーター: 視聴者へのメッセージ
ナレーター: チャンネル誘導「いいねとチャンネル登録よろしくお願いします！」
```

**タイトル生成ルール**（勝ちパターンを必ず適用）:
- 感情ワードを前半に（【ゾッとした】【知らなかった】【後悔した】など）
- 「40代」直接訴求を含める
- 「損失回避」フレーム（やって後悔・知らなくて損など）
- サムネ上段: 感情ワード（6文字以内）
- サムネ下段: ターゲット+行動説明（15文字以内）

### 4. TSV出力

以下のTSVフォーマットで出力する（タブ区切り・ヘッダー行あり）:

```
話者\t本文\tSE
ナレーター\t（セリフ）\t
イチコ\t（セリフ）\tSE1
スレ民1\t（セリフ）\tSE2
（シーン転換行）\t（シーン転換）\tシーン転換
```

SE列のルール:
- ナレーター → 空欄
- イチコ（スレ主） → `SE1`
- スレ民（奇数エピソード）→ `SE1` / `SE2` を交互
- シーン転換行 → 話者=空欄、本文=`（シーン転換）`、SE=`シーン転換`
