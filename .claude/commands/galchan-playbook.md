## 手順

### 1. devサーバー確認
- `localhost:3001` が起動しているか確認
- 起動していなければ `npm run dev` で起動

### 2. Playbook生成

**プレビュー（保存なし）:**
`GET http://localhost:3001/api/youtube/playbook`

**Obsidianに保存:**
`GET http://localhost:3001/api/youtube/playbook?save=true`

### 3. 結果の解釈

- **成功パターン**: CTR 5%以上 / 維持率 40%以上の動画の共通点を分析
- **失敗パターン**: CTR 2%以下 / 維持率 20%以下の動画の共通点を分析
- **テーマ別成績**: テーマタイプごとの平均CTR・維持率を集計

### 4. 勝ちパターン統合版との照合（必須）

Playbook生成後、以下のファイルと照合して整合性を確認する:

1. `DB/rules/勝ちパターン.md`（直接Readで読み込み）
2. `DB/logs/運用知見.md`

> **ベースパス:** `C:\Users\meiek\Dropbox\アプリ\remotely-save\obsidian\02_youtube\ガルちゃんねる\`

照合チェック:
- [ ] Playbookの成功パターンが勝ちパターン統合版と整合しているか
- [ ] 新たに発見された成功/失敗パターンがあれば、勝ちパターン統合版の更新を提案
- [ ] 運用知見の知見と矛盾していないか

### 5. 活用方法

- `/galchan-ideas` でネタ出しする際に、強いテーマタイプを優先
- `/galchan-script` で台本生成する際に、成功パターンの要素を取り入れる
- 失敗パターンに該当するテーマ・構成は避ける

### 5. 前提条件

- video-history に metrics_fetched: true のエントリが必要
- エントリが少ない段階では「データ蓄積中」と表示される
- テーマタイプが `-` の動画はテーマ別集計に含まれない → 手動でテーマを設定すると精度が上がる

## 完走後メタチェック（自動実行）

> スキル完走後に自動で実行する軽量チェック。問題があればユーザーに報告する。

### チェック項目
1. **ルールとの矛盾**: 今回の実行中にDB/rules/のルールと矛盾する判断をしなかったか？
2. **ルールの不足**: 今回の実行で「こういうルールがあれば迷わなかった」という場面はなかったか？
3. **ログ→ルール昇格候補**: DB/logs/運用知見.mdの知見を今回も参照した場合、ルール化すべきほど頻繁に使われていないか？
4. **数値基準のズレ**: 今回の実行結果が、DB/rules/の数値基準と大きくズレていないか？

### 報告フォーマット
問題がなければ報告不要（サイレント）。問題があった場合のみ：

```
⚠️ メタチェック検出:
- [矛盾/不足/昇格候補/基準ズレ]: 内容
- 推奨アクション: ...
```

→ ユーザーが承認したらDB/rules/を更新する。承認なしに更新しない。
