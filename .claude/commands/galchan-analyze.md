### 2. ディレクターの方針決定

既存分析を読んだ上で、以下を決定してからエージェントに指示を出す：
- **前回分析からの変化点**（何を更新すべきか）
- **各エージェントが重点的に調べるべきポイント**
- **前回のAポジションがまだ有効かの確認指示**

### 3. 分析フェーズ（3エージェント並列）

**Agent tool で以下を同時起動。各エージェントにディレクターの方針メモを渡す：**

#### Agent A: 自チャンネル分析官
**読み込み:** アナリティクス.md + 動画管理リスト.md + 前回の自チャンネル分析レポート
以下を分析：
- 動画別パフォーマンス（再生数/CTR/視聴時間/登録者増の順位）
- **前回からの変化**（伸びた動画・落ちた動画）
- タイトル・サムネの効果（変更前後の比較があれば）
- 視聴者の傾向（コメント内容・高評価率）
- 投稿頻度と再生数の相関
→ JSON形式で分析結果を返す

#### Agent B: 競合スカウト
**読み込み:** 競合チャンネルリスト.md + 前回の競合分析 + 勝ちパターン統合版（Aポジション部分）
競合チャンネルリスト.mdの上位5チャンネルについて、WebSearchで最新動画を検索：
- 直近1ヶ月の新着動画タイトル・再生数
- バズ動画（10万再生以上）の共通パターン
- 新しく参入したチャンネルの有無
- **前回のAポジション（未開拓領域）がまだ空いているか**
→ JSON形式で分析結果を返す

#### Agent C: トレンドウォッチャー
**読み込み:** 勝ちパターン統合版（テーマパターン部分）+ ディレクターの方針メモ
WebSearchで以下を調査：
- 「ガルちゃん まとめ YouTube」直近のトレンド
- X（Twitter）で「ガルちゃんまとめ」関連のバズ
- 40代女性向けYouTubeの新しいフォーマット・手法
- **競合が新たに参入し始めたテーマ**（Aポジションの陳腐化チェック）
→ JSON形式で分析結果を返す

### 4. 統合分析（ディレクター）

3エージェントの結果を**既存分析との差分**として統合：
- **自チャンネルの強み/弱み（前回からの変化）**
- **競合との差別化ポジション（更新が必要か）**
- **Aポジションの有効性チェック結果**
- **新たに見つかったAポジション**
- **取り入れるべき新手法**
- **次に作るべきテーマの方向性**
