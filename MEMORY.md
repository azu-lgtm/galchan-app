## 3/25の問題点と原因・対策（次回必読）

### 問題1: トーンが説明的（最重大）
- **原因**: done台本・競合文字起こしをStep 0で読んだが、生成時に文体を模倣しなかった。数値・機関名を主役にして解説記事風に書いてしまった
- **対策**: 生成前にdone台本の任意の20行を読み直し、「体験談が主役・数値は補足」のトーンを確認してから書く。1発言=1エピソードが原則
- **正**: 「ファミマで毎朝110円のチョコチップメロンパン買うのが日課だった」
- **誤**: 「WHOが世界からトランス脂肪酸を排除しようって目標出してた」

### 問題2: 行数不足（135行 vs 目標250+）
- **原因**: 体験談ベースで書くと1発言が具体的になり行数が減る。Pythonで短い行を機械的に拡張したら定型フレーズが重複した
- **対策**: 機械的拡張は禁止。トピックごとに体験談の数を増やす（1トピック=体験3-4個×反応2-3個=12-14行）

### 問題3: 権威機関名の過多（10回超 vs ルール2-3回）
- **原因**: 各トピックで機関名を出してしまった
- **対策**: 機関名は冒頭1回+本文で1-2回のみ。あとは「国が」「研究で」等で包括
